
1. The Watchdog System (g.js)
This is the primary "guardian" of the bot. It uses a Child Process architecture to monitor the main bot script (zao.js).
 * Function: It launches zao.js. If zao.js crashes due to an error, a system failure, or an intentional stop, the Watchdog detects the closure immediately.
 * Self-Healing: It waits exactly 3 seconds and then automatically restarts the bot. This ensures that a single code error doesn't take the bot down permanently.
2. Authentication & Login (login.js)
The system is highly flexible in how it logs into Facebook. It reads from a file called account.txt and can automatically detect several different formats:
 * AppStates (JSON Array): The standard modern way to save sessions.
 * Cookies: Supports both standard Cookie Strings and the older Netscape format.
 * Credentials: If cookies are missing, it can fallback to Email + Password.
 * Tokens: Direct EAAB/EAAA access tokens.
3. Auto-Relogin & Session Protection
If the Facebook session (AppState/Cookie) expires or becomes invalid, the autoRelogin module takes over:
 * Detection: It monitors for "Session Expired" signals.
 * Retry Logic: It attempts to log back in up to 3 times.
 * Cooldown: Between failed attempts, it waits 3 minutes to avoid being flagged as "spam" or "suspicious activity" by Facebook security.
 * Security Alert: If all 3 attempts fail, it stops and requests manual intervention to prevent the account from being locked.
4. Keep-Alive & External Monitoring
To prevent the bot from "sleeping" (especially on hosting platforms like Railway or Koyeb), it uses two methods:
 * Internal Ping: It pings Facebook every 8–18 minutes to keep the session active and prevent a timeout.
 * External Uptime: It provides an HTTP endpoint (usually on port 3000). You can link this to a service like UptimeRobot. If the bot is healthy, it returns a "status": "success" JSON; if not, it reports an error.
Summary of Priority Actions
| Event | Responsible Module | Action / Wait Time |
|---|---|---|
| Bot Crashes | WATCHDOG | Restarts after 3 seconds |
| Session Expires | Auto Relogin | Immediate retry (3 attempts max) |
| Session Active | Ping + Auto-Save | Every 8–18 minutes / 6 hours |
| Connection Error | handlerWhenListenHasError | Immediate restart |
| Settings Changed | File Watcher | Immediate update |
This setup is quite robust for a mobile-developed bot, as it handles the most common "death" scenarios for Messenger bots automatically.
